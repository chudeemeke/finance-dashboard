Testing the Application

  1. Basic Setup:
    • Place all files in your web server root
    • Ensure HTTPS is enabled (required for PWA/WebAuthn)
    • Open index.html in a modern browser
  2. First Run:
    • Default login: username "admin", password "admin123"
    • The app will create initial data in localStorage/IndexedDB
    • Service Worker will register automatically
  3. Test Features:
    • Try importing a CSV file with transactions
    • Test offline mode by disconnecting internet
    • Set up 2FA in Security settings
    • Enable biometric authentication (if supported)

Browser Requirements

  • Chrome/Edge 90+ (recommended)
  • Firefox 88+
  • Safari 14+
  • Features required: Service Workers, IndexedDB, Web Crypto API
Note: Some features like WebAuthn biometric authentication require a secure context (HTTPS) and compatible hardware.

Troubleshooting

Common Issues:

  1. Service Worker not registering:
    • Ensure site is served over HTTPS
    • Check that sw.js is in the root directory
    • Clear browser cache and reload
  2. Import features not working:
    • Libraries are loaded dynamically
    • Check console for CORS errors
    • Ensure file formats match (CSV, Excel, etc.)
  3. AI features not initializing:
    • TensorFlow.js requires time to load
    • Need sufficient transaction data for training
    • Check console for initialization messages
